Rethinking Grid Systems

Flexbox will completely change the way you think about grids systems.

By far, the most common use case for grids in Web developement is to get two pieces of content side by side. Amost every grid system in existence offers endless combinations of ratios and positions, but really you just want to have a few pieces of content in the same horizontal space, spaced evenly.

With flexbox you pretty much get this by default. A flexbox based grid layout can be as simple as the following HTML and CSS:

<div class="Grid">
  <div class="Grid-cell">...</div>
  <div class="Grid-cell">...</div>
  <div class="Grid-cell">...</div>
</div>
.Grid {
  display: flex;
}

.Grid-cell {
  flex: 1;
}

The above two rules allow you to have a grid row with as many columns as you want, and they'll all be equally wide and equally tall by default.

Almost every CSS framework out there has its own grid system, but all of them have to compromise in some form or another. No grid system has equal height columns, and only a few of them have any kind of vertical alignment.

The two main methods of laying out grid cells are to either use float or inline-block, both have which have their disadvantages.

Floats require clearing which has a whole host of layout issues especially when grid items are within non-floated content sections on a page with a floated sidebar. In addition, clearing floats often necessitates taking up at least one but usually both of the before/after pseduo-elements.

Inline block layouts must address the problem of white-space between inline-block items, and all of the solutions to that problem are hacky and annoying.

Flexbox basically eliminates all of the headaches.

The HTML

<div class="Grid">
  <div class="Grid-cell u-size1of3">...</div>
  <div class="Grid-cell u-size2of3">...</div>
</div>

And the CSS

.Grid, .Grid-cell {
  display: flex;
}

.Grid-cell {
  flex: 0 0 auto;
}

Note: for the grid cell widths I'm using the dimension utility classes from the suit.css library.
