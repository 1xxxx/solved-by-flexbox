Introduction

CSS has been lacking proper layout mechanisms for far too long. Transitions, animations, filters, all of these are great, but they don't address the problems that Web developers have been complaining about for what seems like an eternity.

With Flexbox, we finally have a solution.

This site is dedicated to showcasing problems once hard or impossible to solve with CSS alone, now made trivially easy with Flexbox. And with Internet Explorer 11 and Safari 6.1 now in beta, Flexbox will soon be available in every modern browser.

Check out the demos below. View the styles in the Web inspector or dive into the source to see just how easy CSS layout will become once Flexbox hits mainstream.

Caveats and Known Issues

  • IE10 has some flexbox support, but for an older version of the current spec (display:flexbox).
  • Safari 6 and below support the older and now obsolete syntax (display:box).
  • Firefox does not support multi-line flexboxes. See this bug for more details.
  • Opera, IE, and Safari still require vendor prefixes. See caniuse.com for more details.

About the Code

In addition to showcasing the power of Flexbox, this site also aims to be an example of the CSS design and architectual practices I've long promoted. Namely, modular, reuseble style rules built on a predicatble and testable naming convention.

All of the CSS code in these examples make use of the SUIT naming conventions developed by Nicolas Gallagher and based on OOCSS and BEM methodologies. Each example includes one or more reuseable CSS components allowing you to make use of these patterns in your own projects. Links are provided to their respective components on each example page.

This site also uses the excellent autoprefixer tool by Andrey Sitnik, which takes care of vendor prefixing my CSS so I don't have to think about it.

If you'd like to suggest an additional Flexbox example or contribute your own, feel free to open an issue or submit pull request on Github.
