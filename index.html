Introduction

CSS has been lacking proper layout mechanisms for far too long. Transitions, animations, filters, all of these are great, but they don't address the problems that Web developers have been complaining about for what seems like an eternity. Finally, we have a solution.

This site is dedicated to showcasing problems once hard or impossible to solve with CSS alone, now made trivially easy with Flexbox. With the recent release of iOS 7 and Internet Explorer 11 coming in the fall, the CSS Flexible Box Model, i.e. Flexbox, has finally hit mainstream.

Check out the demos below to see just how easy your life will be with Flexbox.

Caveats and Known Issues

  • IE10 has flexbox support but only for an older version of the current spec (display:flexbox).
  • Safari 6 and below support the original but now obsolete syntax (display:box).
  • Firefox does not support multi-line flexboxes. See this bug for more details.
  • Opera, IE, and Safari still require vendor prefixes. See caniuse.com for more details.
