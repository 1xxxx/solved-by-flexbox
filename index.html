Introduction

CSS has been lacking proper layout mechanisms for far too long. Transitions, animations, filters, all of these are great, but they don't address the problems that Web developers have been complaining about for what seems like an eternity. Finally, with the Flexible Box Model module (a.k.a Flexbox), we have a solution.

This site is dedicated to showcasing problems once hard or impossible to solve with CSS alone, now made trivially easy with Flexbox. And with Internet Explorer 11 and Safari 6.1 now in beta, Flexbox will soon be available in every modern browser.

Take a look at the demos below, open up the Web inspector, and dive into the source to see just how easy CSS layout will become once Flexbox hits mainstream.

Caveats and Known Issues

  • IE10 has some flexbox support, but for an older version of the current spec (display:flexbox).
  • Safari 6 and below support the older and now obsolete syntax (display:box).
  • Firefox does not support multi-line flexboxes. See this bug for more details.
  • Opera, IE, and Safari still require vendor prefixes. See caniuse.com for more details.

About this Project

In addition to showcasing the power of Flexbox, this site also aims to promote good CSS design and architecture.

All of the examples showcases, as well as the site design itself follows the suit.css design principles, which are based off of OOCSS and BEM methodologies.
